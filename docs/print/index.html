Tutorial GTK

¿Qué es GTK?

  • Un toolkit (kit de utilerías) para crear interfaces de usuario orientado a objetos escrito en C.
  • También conocido como GIMP Toolkit.
  • Proporciona 186 componentes de control gráfico (Widgets).
  • El release inicial fue el 14 de Abril de 1998.
  • Adoptado por GNOME para desarrollar su entorno gráfico.
  • Licenciado bajo los términos GNU LGPL.

Conceptos

  • Todas las aplicaciones de GTK consisten de una jerarquía de Widgets.
  • El widget principal de cada aplicación es Window.
  • GTK es guiado por Eventos. Cuando ocurre un evento se notifica a un Escuchador (función).
  • Las Señales conectan un evento específico de la aplicación con múltiples escuchadores (funciones).

Convenciones

Todas los nombres de funciones tienen la forma siguiente:

gtk_window_set_title(instancia, ...parametros)

  • Librería: gtk | gdk | g (espacio de nombres)
  • Clase: window | object | button | ...
  • Método (getters, setters, etc.)
  • Instancia
#include <gtk/gtk.h>
​
static void activate(GtkApplication* app, gpointer user_data) {
  GtkWidget *window = gtk_application_window_new(app);
​
  gtk_window_set_title (GTK_WINDOW(window), "Window");
  gtk_window_set_default_size (GTK_WINDOW(window), 200, 200);
  gtk_widget_show_all(window);
}
​
int main(int argc, char **argv) {
  GtkApplication *app = gtk_application_new("org.gtk.example", G_APPLICATION_FLAGS_NONE);
​
  g_signal_connect(app, "activate", G_CALLBACK(activate), NULL);
  int status = g_application_run(G_APPLICATION(app), argc, argv);
  g_object_unref(app);
​
  return status;
}

static void activate(GtkApplication* app, gpointer user_data) {
  GtkWidget *window = gtk_application_window_new(app);
​
  GtkWidget *contenedor = gtk_fixed_new();
  gtk_container_add(GTK_CONTAINER(window), contenedor);
​
  GtkWidget *imagen = gtk_image_new_from_file("Imagen.gif");
  gtk_fixed_put(GTK_FIXED(contenedor), imagen, 0, 0);
​
  gtk_window_set_title (GTK_WINDOW(window), "Window");
  gtk_window_set_default_size (GTK_WINDOW(window), 200, 200);

Layouts

  • Un Layout define una estrategia para colocar elementos en pantalla.
  • Fixed: Coloca elementos en una coordenada fija.
  • Box: Coloca elementos en una fila o una columna.
  • Grid: Coloca elementos en una cuadrícula.

Live Coding ⌨️

Concurrencia

  • Un proceso no puede acceder directamente a las variables de otro proceso.
  • El proceso principal de la interfaz de usuario no puede ser bloqueado.

Hilos

  • Son procesos ligeros que comparten el espacio de memoria, archivos abiertos, etc.
  • Se utiliza la librería pthread.
